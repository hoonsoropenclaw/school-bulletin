# 設計決策日誌 (DESIGN DECISIONS LOG)

> 任何 PRD / 受眾邏輯 / 角色矩陣 / UI 設計的「重大決策」都記在這裡。
> **目的**: 未來任何人（含赫米斯自己、其他 AI、未來的自己）能快速理解「為什麼目前是這樣」、避免「為什麼當初這樣做？改成另一個版本吧」的循環。
> **格式**: 每一筆決策 7 個欄位（時間 / 觸發 / 上一版 / 這版 / 為什麼 / 預期 E2E / 決策者）

---

## 決策 001 — matchAudience 受眾匹配規則 v4（最終定案）

- **時間**: 2026-06-11
- **觸發**: 使用者回饋「teacher_lin、parent_chen 都應該要看到 4 個公告、設定帳號登入的目的在於看到訪客看不到的『內部公告』、所以在『內部公告』這個機制完成之前、任何人都應該看到全部的『外部公告』」
- **上一版**: v3（訪客看公開、處室看全部、受眾看 audience+公開）
- **這版**: v4（**任何角色都看全部**、不分受眾、不分公開）
- **為什麼**:
  - 「登入後能看到的 >= 未登入」是設計聖經
  - 登入的價值 = 看到內部公告、不是看到 audience 命中
  - 在「內部公告」機制未建立前、受眾不該被 audience 過濾
  - 否則登入 = 看更少 = 登入動機消失 = 設計 bug
- **預期 E2E**:
  - 訪客（無 cookie）→ 4 個 ✅
  - 6 個處室帳號 → 4 個 ✅
  - teacher_lin → 4 個 ✅
  - parent_chen → 4 個 ✅
  - student_wang → 4 個 ✅
  - 受眾 POST 公告 → 403 ✅
  - 處室 POST 公告 → 201 ✅
- **決策者**: 使用者（明確指示）
- **未來啟用「內部公告」時**: 啟用 audience_type: 'public' | 'internal' | 'role_specific' 欄位、matchAudience 才分角色

## SOP：新增決策時的格式

```markdown
## 決策 NNN — <一句話決策標題>

- **時間**: YYYY-MM-DD
- **觸發**: 觸發這個決策的具體事件
- **上一版**: 改成這個版本之前的邏輯
- **這版**: 新決策的內容
- **為什麼**: 1-3 個原因、必須包含「登入後 >= 未登入」聖經檢查（如果適用）
- **預期 E2E**: 預期的 E2E 驗證結果（具體角色、具體公告數）
- **決策者**: 使用者 / 赫米斯 / 共同討論
```

**If** 改任何受眾邏輯、角色矩陣、權限設計 **Then** 必先寫一筆決策到這裡、再改 code
**If** 決策被後續決策取代 **Then** 把這筆標 `（已廢棄、被 NNN 取代）`、不刪除（保留 audit trail）
